## Overview

The Unlingo API provides programmatic access to your translations. Use it to fetch translations for your applications, integrate with CI/CD pipelines, or build custom tooling.

## Base URL

All API requests should be made to:

```
https://api.unlingo.com/v1
```

## Authentication

All API endpoints require authentication using an API key. You can create API keys from your project settings in the dashboard.

<CardGroup cols={2}>
    <Card title='Header Authentication' icon='key'>
        Pass your API key in the `x-api-key` header
    </Card>
    <Card title='Bearer Token' icon='shield'>
        Use `Authorization: Bearer YOUR_API_KEY`
    </Card>
</CardGroup>

### Header Authentication

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.unlingo.com/v1/translations?..."
```

### Bearer Token

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  "https://api.unlingo.com/v1/translations?..."
```

<Tip>
    API keys are scoped to individual projects. Create separate keys for development, staging, and production environments.
</Tip>

## Available Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/translations` | GET | Fetch translations for a release |
| `/v1/health` | GET | Check API status |

## Rate Limiting

API requests are counted against your subscription plan's monthly limit:

| Plan | Monthly Requests |
|------|-----------------|
| Starter | 10,000 |
| Hobby | 50,000 |
| Premium | 250,000+ |

### Rate Limit Response

When approaching limits, responses include warning headers:

```
X-RateLimit-Warning: Approaching limit (80%)
X-RateLimit-Remaining: 2000
```

When exceeded (130% of limit), requests return `429 Too Many Requests`:

```json
{
  "error": "Rate limit exceeded",
  "message": "Monthly API request limit exceeded. Upgrade your plan or wait for reset."
}
```

## Response Format

All responses are JSON formatted:

### Success Response

```json
{
  "welcome": "Hello",
  "goodbye": "Goodbye"
}
```

### Error Response

```json
{
  "error": "Error type",
  "message": "Human-readable error message"
}
```

## HTTP Status Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 404 | Not Found - Resource doesn't exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

## Error Handling

<CodeGroup>

```javascript JavaScript
try {
  const response = await fetch(
    'https://api.unlingo.com/v1/translations?release=main&namespace=common&lang=en',
    {
      headers: { 'x-api-key': 'YOUR_API_KEY' }
    }
  );

  if (!response.ok) {
    const error = await response.json();
    console.error('API Error:', error.message);
    return;
  }

  const translations = await response.json();
} catch (error) {
  console.error('Network error:', error);
}
```

```typescript TypeScript
interface ApiError {
  error: string;
  message: string;
}

async function fetchTranslations(): Promise<Record<string, string> | null> {
  const response = await fetch(
    'https://api.unlingo.com/v1/translations?release=main&namespace=common&lang=en',
    {
      headers: { 'x-api-key': process.env.UNLINGO_API_KEY! }
    }
  );

  if (!response.ok) {
    const error: ApiError = await response.json();
    throw new Error(error.message);
  }

  return response.json();
}
```

</CodeGroup>

## Best Practices

### Caching

<Tip>
    Implement client-side caching to reduce API calls and stay within rate limits.
</Tip>

```javascript
// Cache translations for 1 hour
const CACHE_DURATION = 60 * 60 * 1000;
let cachedTranslations = null;
let cacheTimestamp = 0;

async function getTranslations() {
  if (cachedTranslations && Date.now() - cacheTimestamp < CACHE_DURATION) {
    return cachedTranslations;
  }

  const translations = await fetchFromApi();
  cachedTranslations = translations;
  cacheTimestamp = Date.now();
  return translations;
}
```

### Error Fallbacks

Always have fallback translations:

```javascript
async function loadTranslations(lang) {
  try {
    return await fetchTranslations(lang);
  } catch (error) {
    console.warn('Failed to load translations, using fallback');
    return fallbackTranslations;
  }
}
```

### Environment Variables

Never hardcode API keys:

```bash
# .env
UNLINGO_API_KEY=your_api_key_here
```

```javascript
const apiKey = process.env.UNLINGO_API_KEY;
```

<Danger>
    Never commit API keys to version control. Use environment variables and secrets management.
</Danger>
