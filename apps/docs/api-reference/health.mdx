## Overview

The health check endpoint allows you to verify that the Unlingo API is operational. Use this for monitoring, CI/CD pipelines, or application startup checks.

## Authentication

<ParamField header='x-api-key' type='string' required>
    Your API key for authentication
</ParamField>

<ParamField header='Authorization' type='string'>
    Alternative: Bearer token with your API key
</ParamField>

## Example Request

<RequestExample>

```bash cURL
curl -X GET "https://api.unlingo.com/v1/health" \
  -H "x-api-key: YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.unlingo.com/v1/health', {
    headers: {
        'x-api-key': 'YOUR_API_KEY',
    },
});

const status = await response.json();
console.log('API Status:', status);
```

```typescript TypeScript
interface HealthResponse {
  status: 'ok' | 'error';
  timestamp?: string;
}

async function checkHealth(): Promise<HealthResponse> {
  const response = await fetch('https://api.unlingo.com/v1/health', {
    headers: {
      'x-api-key': process.env.UNLINGO_API_KEY!,
    },
  });

  return response.json();
}
```

</RequestExample>

## Status Codes

| Code | Description |
|------|-------------|
| 200 | API is healthy and operational |
| 401 | Invalid or missing API key |
| 503 | Service temporarily unavailable |

## Use Cases

### Application Startup

Check API availability before your application fully starts:

```javascript
async function initializeApp() {
  const healthCheck = await fetch('https://api.unlingo.com/v1/health', {
    headers: { 'x-api-key': process.env.UNLINGO_API_KEY }
  });

  if (!healthCheck.ok) {
    console.warn('Unlingo API unavailable, using cached translations');
    return loadCachedTranslations();
  }

  return loadFreshTranslations();
}
```

### Monitoring

Integrate with monitoring tools to track API availability:

```javascript
// Check every 5 minutes
setInterval(async () => {
  try {
    const response = await fetch('https://api.unlingo.com/v1/health', {
      headers: { 'x-api-key': process.env.UNLINGO_API_KEY }
    });

    if (response.ok) {
      metrics.recordHealthy('unlingo-api');
    } else {
      metrics.recordUnhealthy('unlingo-api', response.status);
    }
  } catch (error) {
    metrics.recordUnhealthy('unlingo-api', 'network-error');
  }
}, 5 * 60 * 1000);
```

### CI/CD Pipeline

Verify API connectivity in deployment pipelines:

```yaml
# GitHub Actions example
- name: Check Unlingo API
  run: |
    response=$(curl -s -o /dev/null -w "%{http_code}" \
      -H "x-api-key: ${{ secrets.UNLINGO_API_KEY }}" \
      "https://api.unlingo.com/v1/health")

    if [ "$response" != "200" ]; then
      echo "Unlingo API health check failed with status $response"
      exit 1
    fi
    echo "Unlingo API is healthy"
```

<Tip>
    Health checks still count against your API rate limit. Use them sparingly in monitoring scenarios.
</Tip>
