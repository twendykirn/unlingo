## Overview

The Unlingo API is a RESTful API that allows you to programmatically manage your translation projects, namespaces, versions, and retrieve translations for your applications. All API requests use HTTPS and return JSON responses.

## Base URL

All API requests should be made to:

```
https://api.unlingo.com/v1
```

## Authentication

The Unlingo API uses API key authentication. Include your API key in the `Authorization` header:

```bash
Authorization: Bearer YOUR_API_KEY
```

## Request Format

### HTTP Methods

| Method | Usage |
|--------|-------|
| `GET` | Retrieve data |
| `POST` | Create new resources |
| `PUT` | Update existing resources |
| `DELETE` | Remove resources |

### Content Type

All requests that include a request body should use:

```
Content-Type: application/json
```

### Example Request

```bash
curl -X GET "https://api.unlingo.com/v1/translations?version=1.0.0&namespace=common&lang=en" \
  -H "Authorization: Bearer ul_1234567890abcdef" \
  -H "Content-Type: application/json"
```

## Status Codes

The API uses standard HTTP status codes:

| Code | Status | Description |
|------|--------|-------------|
| `200` | OK | Request successful |
| `201` | Created | Resource created successfully |
| `400` | Bad Request | Invalid request parameters |
| `401` | Unauthorized | Invalid or missing API key |
| `403` | Forbidden | Insufficient permissions |
| `404` | Not Found | Resource not found |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server error |

## Rate Limiting

API requests are rate limited based on your plan:

| Plan | Requests per minute | Requests per hour |
|------|-------------------|------------------|
| Free | 100 | 1,000 |
| Pro | 1,000 | 10,000 |
| Enterprise | Custom | Custom |

When you exceed the rate limit, you'll receive a `429` status code with headers indicating when you can retry:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1642248600
Retry-After: 60
```

## Pagination

List endpoints support pagination using cursor-based pagination:

### Request Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| `limit` | Number of items per page (max 100) | 20 |
| `cursor` | Pagination cursor from previous response | - |

### Response Format

```json
{
  "data": [
    // Array of items
  ],
  "pagination": {
    "hasMore": true,
    "nextCursor": "eyJpZCI6IjEyMzQ1NiJ9",
    "limit": 20
  }
}
```

### Example

```bash
# First page
curl "https://api.unlingo.com/v1/projects?limit=20"

# Next page
curl "https://api.unlingo.com/v1/projects?limit=20&cursor=eyJpZCI6IjEyMzQ1NiJ9"
```

## Filtering and Sorting

Many list endpoints support filtering and sorting:

### Filtering

```bash
# Filter projects by name
curl "https://api.unlingo.com/v1/projects?name=my-app"

# Filter by multiple criteria
curl "https://api.unlingo.com/v1/namespaces?project=proj_123&status=active"
```

### Sorting

```bash
# Sort by creation date (newest first)
curl "https://api.unlingo.com/v1/projects?sort=-createdAt"

# Sort by name (ascending)
curl "https://api.unlingo.com/v1/projects?sort=name"
```

Available sort fields vary by endpoint. Use `-` prefix for descending order.

## Versioning

The API version is included in the URL path. The current version is `v1`.

Breaking changes will result in a new API version. We'll maintain backward compatibility for at least 12 months after releasing a new version.

## Error Codes

### Common Error Codes

| Code | Description |
|------|-------------|
| `INVALID_API_KEY` | The provided API key is invalid |
| `INSUFFICIENT_PERMISSIONS` | API key lacks required permissions |
| `RESOURCE_NOT_FOUND` | The requested resource doesn't exist |
| `VALIDATION_ERROR` | Request validation failed |
| `RATE_LIMIT_EXCEEDED` | Too many requests |
| `NAMESPACE_NOT_FOUND` | Specified namespace doesn't exist |
| `VERSION_NOT_FOUND` | Specified version doesn't exist |
| `LANGUAGE_NOT_SUPPORTED` | Requested language is not available |

### Handling Errors

```javascript
async function handleApiRequest(url, options) {
  try {
    const response = await fetch(url, options);
    
    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(`API Error: ${errorData.error.message}`);
    }
    
    const data = await response.json();
    return data.data; // Return just the data portion
  } catch (error) {
    console.error('API request failed:', error);
    throw error;
  }
}
```

## Idempotency

POST and PUT requests support idempotency to prevent duplicate operations. Include an `Idempotency-Key` header with a unique value:

```bash
curl -X POST "https://api.unlingo.com/v1/projects" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Idempotency-Key: unique-key-123" \
  -H "Content-Type: application/json" \
  -d '{"name": "My Project"}'
```

If you make the same request with the same idempotency key within 24 hours, you'll receive the same response without creating a duplicate resource.

## Webhooks

Unlingo supports webhooks to notify your application of events:

### Supported Events

- `translation.updated` - Translation content changed
- `release.published` - New release published
- `project.created` - New project created
- `namespace.created` - New namespace created

### Webhook Configuration

Set up webhooks in your project settings or via the API:

```bash
curl -X POST "https://api.unlingo.com/v1/webhooks" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.com/webhooks/unlingo",
    "events": ["translation.updated", "release.published"],
    "secret": "your-webhook-secret"
  }'
```

### Webhook Security

Webhooks include a signature header for verification:

```javascript
const crypto = require('crypto');

function verifyWebhookSignature(payload, signature, secret) {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
    
  return `sha256=${expectedSignature}` === signature;
}
```
