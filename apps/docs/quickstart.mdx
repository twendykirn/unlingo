## Overview

This quickstart guide will help you set up Unlingo and integrate it into your application in just a few minutes. We'll create a project, add some translations, and fetch them using our REST API.

## Prerequisites

- An Unlingo account (sign up at [app.unlingo.com](https://app.unlingo.com/sign-up))
- Basic knowledge of REST APIs
- A web or mobile application project

## Step 1: Create Your Project

<Steps>
  <Step title="Sign in to Dashboard">
    Navigate to [app.unlingo.com](https://app.unlingo.com) and sign in to your account.
  </Step>
  <Step title="Create New Project">
    Click "Create Project" and enter your project details:
    - **Name**: Your application name (e.g., "My Web App")
    - **Description**: Brief description of your project
  </Step>
  <Step title="Create Namespace">
    Within your project, create a namespace:
    - **Name**: "common" (for shared translations)
    - **Description**: "Common UI translations"
  </Step>
</Steps>

## Step 2: Add Your Translations

<Tabs>
  <Tab title="Dashboard (Recommended)">
    1. Navigate to your namespace
    2. Click "Add Translation Key"
    3. Add some sample keys:

    ```json
    {
      "welcome": "Welcome to our application",
      "login": "Sign In",
      "logout": "Sign Out",
      "nav": {
        "home": "Home",
        "about": "About",
        "contact": "Contact"
      }
    }
    ```
  </Tab>
  <Tab title="API Upload">
    Use our API to upload translations programmatically:

    ```bash
    curl -X POST https://api.unlingo.com/v1/translations \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "namespace": "common",
        "language": "en",
        "translations": {
          "welcome": "Welcome to our application",
          "login": "Sign In"
        }
      }'
    ```
  </Tab>
</Tabs>

## Step 3: Create Your First Release

<Steps>
  <Step title="Navigate to Releases">
    Go to the "Releases" tab in your project dashboard
  </Step>
  <Step title="Create Release">
    Click "Create Release" and enter:
    - **Version**: "1.0.0"
    - **Description**: "Initial release"
    - **Namespaces**: Select your "common" namespace
  </Step>
  <Step title="Publish">
    Click "Publish Release" to make it available via the API
  </Step>
</Steps>

## Step 4: Get Your API Key

<Steps>
  <Step title="Navigate to API Keys">
    Go to the "API Keys" tab in your project settings
  </Step>
  <Step title="Create API Key">
    Click "Create API Key" and give it a name like "Production API Key"
  </Step>
  <Step title="Copy Key">
    Copy the generated API key - you'll need it for the next step
  </Step>
</Steps>

<Warning>
  Keep your API key secure and never expose it in client-side code. For frontend applications, consider using environment variables or a backend proxy.
</Warning>

## Step 5: Fetch Translations

Now you can fetch your translations using our API:

<CodeGroup>

```bash cURL
curl -X GET "https://api.unlingo.com/v1/translations?version=1.0.0&namespace=common&lang=en" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript/Fetch
const response = await fetch(
  'https://api.unlingo.com/v1/translations?version=1.0.0&namespace=common&lang=en',
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const translations = await response.json();
console.log(translations);
```

```javascript Node.js
const axios = require('axios');

async function getTranslations() {
  try {
    const response = await axios.get(
      'https://api.unlingo.com/v1/translations',
      {
        params: {
          version: '1.0.0',
          namespace: 'common',
          lang: 'en'
        },
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY'
        }
      }
    );
    
    return response.data;
  } catch (error) {
    console.error('Error fetching translations:', error);
  }
}
```

```python Python
import requests

url = "https://api.unlingo.com/v1/translations"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}
params = {
    "version": "1.0.0",
    "namespace": "common",
    "lang": "en"
}

response = requests.get(url, headers=headers, params=params)
translations = response.json()
print(translations)
```

</CodeGroup>

## Common Patterns

### Environment-Based Versions

Use different versions for different environments:

```javascript
const version = process.env.NODE_ENV === 'production' ? '1.0.0' : 'staging';

const translations = await fetch(
  `https://api.unlingo.com/v1/translations?version=${version}&namespace=common&lang=en`,
  {
    headers: {
      'Authorization': `Bearer ${process.env.UNLINGO_API_KEY}`
    }
  }
);
```

### Multiple Languages

Fetch translations for multiple languages:

```javascript
const languages = ['en', 'es', 'fr'];
const translations = {};

for (const lang of languages) {
  const response = await fetch(
    `https://api.unlingo.com/v1/translations?version=1.0.0&namespace=common&lang=${lang}`,
    {
      headers: {
        'Authorization': `Bearer ${process.env.UNLINGO_API_KEY}`
      }
    }
  );
  
  translations[lang] = await response.json();
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="API Key not working">
    Make sure your API key is correct and has the proper permissions. Check that you're using the correct project ID and that the API key hasn't expired.
  </Accordion>
  
  <Accordion title="Translations not found">
    Verify that:
    - Your release is published
    - The version, namespace, and language parameters are correct
    - Your translations were saved properly in the dashboard
  </Accordion>
  
  <Accordion title="CORS errors in browser">
    Our API includes CORS headers for browser requests. If you're still seeing errors, consider using a backend proxy to fetch translations server-side.
  </Accordion>
</AccordionGroup>
